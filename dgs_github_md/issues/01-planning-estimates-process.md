# [Planning] Create project plan, estimates and daily process documentation

## Goal
Create the project planning required by the assignment before building features.

## Why
The PDF requires a task breakdown, estimates, actual time tracking and documentation of process for the report.

## Tasks
- [ ] Use GitHub Issues as the main todo list.
- [ ] Add an estimate to each issue.
- [ ] Track actual time used for each main issue.
- [ ] Create a daily plan for the project period.
- [ ] Take screenshots of GitHub Issues/Projects for the report.
- [ ] Note blockers, decisions and solutions while working.
- [ ] Keep the report updated during the project, not only at the end.

## Acceptance criteria
- [ ] Issues exist for all mandatory tasks.
- [ ] Planned time and actual time can be documented.
- [ ] Report has enough process material.

## Estimate
1-2 hours initially + 10-15 minutes daily.
